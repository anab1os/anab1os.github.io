📑 Содержание

    📋 Общий гайд по проверке на читы CS2

    Основные положения

    1. Скачиваем все программы только с официальных сайтов. Не предлагайте свои версии.
    2. Создайте отдельную папку для инструментов проверки.
    3. Игнорируйте скрытый рабочий стол и доп. мониторы. Работайте только с основным.

    Типы доказательств

    1. Хранение — файл найден в файловой системе.
    2. Сокрытие — логи удаления/переименования.
    3. Запуск — следы запуска .exe/.dll в памяти или Prefetch (.pf).
    Правило выдачи наказания:
    Чит должен быть либо запущен в течение последних 3 месяцев, либо храниться на ПК. Достаточно признания или характерных признаков (размер, цифровая подпись, троллинг-папка и т.д.).

    Если нашли подозрительный запуск, но файл отсутствует

    1. Ищите по названию в Everything.
    2. Если не нашли — JournalTrace и ищите действия с этим файлом.

    • ✅ Проверка CS2 по фасту (инвентарь, оверлей и т.д.)

    • ✅ Использование всех базовых программ

    • Фильтр Rename: old name
    • Анализ через Show file information

    🖥️ System Informer

    Установка: Включите "Enable kernel-mode driver" в System → Options. При ошибке — Help → Check for Updates → Canary.

    Поиск файлов

    1. Нажмите ctrl + f
    2. Введите специализированный список для этого способа
    3. Обязательно включите RegEx
    4. Нажмите "Искать" (Find)
    5. Просматривайте странные процессы

    Проверка через csrss.exe

    1. Запуск от имени администратора
    2. System → Options → Enable Kernel Mode Driver
    3. Завершить все процессы античитов
    4. Search Processes → csrss.exe
    5. Выбираем один из двух процессов
    6. Properties → Options → Strings → Отмечаем все галочки + ставим минимальную длину 4 → Filter → RegEx → Вставляем текст

    TIER 1 Экспертная проверка

    Уровень для сложных случаев. Применяется, когда методы TIER 3 / TIER 2 не дали результата, но подозрения остаются (аппаратные читы, спуфинг, продвинутое сокрытие).

    Критерии TIER 1

    • ✅ Всё из TIER 2
    • ✅ Анализ полного дампа памяти процессов (cs2.exe, csrss.exe, explorer.exe)
    • ✅ Поиск следов DMA / аппаратных читов
    • ✅ Проверка спуфинга HWID
    • ✅ Анализ драйверов и ядра (kernel-mode)

    🧬 Анализ дампов памяти

    Снимайте полный дамп подозрительных процессов и анализируйте строки, импорты и загруженные модули.

    • Ищите неподписанные .dll, загруженные в cs2.exe.
    • Проверяйте скрытые (unlinked) модули, отсутствующие в списке загруженных.
    • Строки с URL, названиями читов, путями к конфигам.

    🔌 DMA и аппаратные читы

    DMA (Direct Memory Access) — чит читает память через отдельную PCIe/FPGA-плату, программных следов на целевом ПК почти нет. Признаки:

    • 🔴 Нестандартные PCIe-устройства в диспетчере устройств (Screamer, LeetDMA и т.п.).
    • 🔴 Второй ПК / ноутбук рядом, карта захвата изображения (capture card).
    • 🔴 Подозрительные USB-устройства ввода (KMBox, эмуляторы мыши).
    Важно: DMA часто требует физического осмотра рабочего места, а не только ПО.

    🎭 Проверка спуфинга (HWID)

    • Сверяйте серийные номера дисков, MAC-адрес и SMBIOS с реальным «возрастом» системы.
    • 🔴 Свежие идентификаторы при старой дате установки Windows — признак спуфинга.
    • Проверяйте наличие спуфер-утилит в истории запусков (BAM, Prefetch).

    🛡️ Драйверы и ядро

    • Проверяйте список загруженных драйверов на неподписанные / тестовые.
    • 🔴 Включённый Test Mode Windows или отключённая проверка подписи драйверов.
    • Ищите драйверы, загруженные из Temp или пользовательских папок.

    🧠 Анализ CSRSS (csrss.exe)

    csrss.exe — системный процесс Windows (Client Server Runtime Subsystem). Он логирует все возможные запуски программ в текущей сессии. Анализ его памяти позволяет обнаружить следы исполнения .exe и .dll файлов, даже если сами файлы уже удалены.

    🔹 Анализ для .exe файлов

    1. В System Informer выберите процесс csrss.exe с меньшим размером в байтах.
    2. Примените фильтр по regex'у: 
      ^[A-Z]:\\.+\.(exe)$
    3. Создайте папку с любым удобным названием и сохраните дамп найденных строк.
    4. Проанализируйте полученные данные с помощью Paths Parser.

    🔹 Анализ для .dll файлов

    1. Выберите процесс csrss.exe с большим размером в байтах.
    2. Примените фильтр по regex'у: 
      ^[A-Z]:\\.+\.(dll)$
    3. Сохраните дамп и проанализируйте через Paths Parser.

    🔹 Обнаружение .dll и .exe одновременно

    Для отображения обоих типов файлов в одном результате используйте универсальный regex:

    ^[a-zA-Z]:\\.+(\\)?.*?$

    🔍 Поиск файлов с изменёнными расширениями

    ^[a-z]:.+\.((?!exe|pyd|manifest|dll|config|\\|cpl|microsoft-|shell).)*$

    🖥️ PowerShell History (PSReadline)

    Начиная с PowerShell v5 в Windows 10, команды, введенные в консоли PowerShell, будут записываться модулем PSReadline в файл ConsoleHost_history.txt.

    Сессии PowerShell без консоли, такие как содержимое .ps1 скриптов или выполнение команд через PowerShell ISE, не будут записываться в этот файл.

    Расположение:
    %AppData%\Microsoft\Windows\PowerShell\PSReadLine\ConsoleHost_history.txt

    Особенности:

    • Содержит введенные команды, по одной команде на строку, без связанных временных меток (или любых дополнительных метаданных).
    • Временную метку последнего выполнения команды можно определить по временной метке последней записи самого файла.
    • По умолчанию хранятся только последние 4096 команд.

    Обход записи PSReadline:
    Существует необычный способ обхода записи — требуется выгрузка модуля PSReadline (например, с помощью команды Remove-Module PSReadline в существующей сессии PowerShell).

    🛡️ firewall.cpl — Брандмауэр Windows

    firewall.cpl — это классическая панель управления Брандмауэром Windows. Через неё можно мгновенно управлять сетевой защитой без поиска по меню.

    Также через данную панель можно детектить некоторые рейдж-клиенты, по типу Nursultan, Celestial и других.

    Как открыть:

    1. WIN + R
    2. Введите firewall.cpl и нажмите Enter
    3. Перейдите в Дополнительные параметры (Advanced settings)
    4. Выберите Правила для входящих подключений (Inbound Rules)

    Что искать: правила с нестандартными именами, открывающие порты для обхода античит-систем, а также правила, созданные подозрительными приложениями. Часто рейдж-клиенты создают свои собственные правила для обеспечения стабильного соединения с сервером управления.

    Совет: Обратите внимание на правила, у которых «Действие» — Разрешить подключение, а «Протокол» — TCP или UDP с нестандартными номерами портов (например, 4444, 5555, 1337 и т.п.).

    🚩 Красные флаги (признаки возможных читов)

    📁 Файловая система

    • 🔴 Файлы с названиями, содержащими cheat, hack, aim, wall, inject.
    • 🔴 Странные файлы с расширениями .exe или .dll, расположенные в Temp, AppData\Local\Temp, C:\Users\Public.
    • 🔴 Наличие упаковщиков (UPX, VMProtect) в исполняемых файлах.
    • 🔴 Файлы с поддельными цифровыми подписями.
    • 🔴 Конфигурационные файлы читов (.cfg, .ini, .json) с подозрительными названиями.

    🧠 Память и процессы

    • 🔴 Процессы с нестандартными именами, например csgo.exe в папке AppData.
    • 🔴 Процессы, запущенные из папок без прав на запись (например, System32) с подозрительными именами.
    • 🔴 В памяти csrss.exe найдены пути к .exe или .dll, которые не являются системными.
    • 🔴 В памяти игрового процесса (cs2.exe) обнаружены строки с названиями читов или URL-адресами.

    📜 Логи и артефакты

    • 🔴 В JournalTrace зафиксированы массовые удаления .exe или .dll сразу после запуска CS2.
    • 🔴 В Prefetch есть записи о запуске подозрительных .exe, которые сейчас отсутствуют на диске.
    • 🔴 В реестре есть ключи с именами читов (например, Software\Aimware, Software\Vape).
    • 🔴 В ShellBag есть посещённые папки, ведущие на сайты с читами или на папки с названиями типа cheats, injects.
    • 🔴 В BAM Parser есть записи о запуске файлов, которые уже удалены с диска.

    🌐 Сеть

    • 🔴 Активные сетевые соединения на нестандартные порты (например, 4444, 1337, 5555).
    • 🔴 Наличие в памяти строк с IP-адресами, не относящимися к игровым серверам.

    🎯 Типы читов и их признаки

    🎯 Aimbot (аим)

    Автоматическое наведение прицела на противника.

    Признаки: неестественно резкие «доводки», щелчки прицела точно в голову, идеальный трекинг сквозь дым.

    🧱 Wallhack / ESP

    Отображение противников, их HP, оружия и предметов сквозь стены.

    Признаки: префайр по невидимым врагам, реакция на пуш до появления в зоне видимости, взгляд «сквозь» стены.

    ⚡ Triggerbot

    Автоматический выстрел при наведении прицела на врага.

    Признаки: мгновенная реакция (менее 100 мс), стабильные однопульные килы при пересечении прицела.

    🐰 Bhop / Movement

    Идеальные банни-хопы и авто-стрейф.

    Признаки: абсолютно стабильная скорость в серии прыжков, невозможная вручную синхронизация.

    📡 Radar / Spread

    Раскрытие позиций на радаре, контроль разброса.

    Признаки: ротации «на опережение», знание позиций без информации от команды.

    🔌 DMA / Hardware

    Аппаратные читы через отдельное устройство. Самые сложные для обнаружения — см. вкладку TIER 1.

    Признаки: отсутствие программных следов при явной игре с ESP/aim, второй ПК, карта захвата.

    Названия читов полезно знать для поиска по файлам и реестру: Aimware, Neverlose, Vape, Fatality, Osiris, Nixware, Gamesense и др.

    📖 Глоссарий терминов

    🔹 USN Journal (Update Sequence Number Journal)

    Журнал изменений файловой системы NTFS. Фиксирует все изменения (создание, удаление, переименование, изменение атрибутов). Используется для восстановления истории файлов даже после их удаления.

    🔹 MFT (Master File Table)

    Главная таблица файлов в NTFS. Содержит метаданные каждого файла и папки на томе. Анализ MFT позволяет находить скрытые или удалённые записи.

    🔹 ShellBags

    Записи в реестре Windows, хранящие историю открытия папок через проводник (размер окна, положение, представление). Могут сохранять информацию о папках, которые уже удалены.

    🔹 Prefetch

    Механизм Windows для ускорения загрузки приложений. Хранит информацию о запущенных .exe (путь, количество запусков, дата, загруженные модули). Находится в C:\Windows\Prefetch\*.pf.

    🔹 BAM (Background Activity Moderator)

    Артефакт реестра (HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\bam\UserSettings). Хранит время последнего запуска каждого .exe, даже если файл уже удалён.

    🔹 CSRSS (Client Server Runtime Subsystem)

    Системный процесс Windows, отвечающий за управление консольными окнами и завершением процессов. В его памяти могут оставаться строки с путями к запущенным приложениям.

    🔹 Инжект (Code Injection)

    Внедрение кода (обычно DLL) в адресное пространство другого процесса. Читы часто используют инжект для модификации игровой памяти.

    🔹 Спуфинг (Spoofing)

    Подмена аппаратных идентификаторов (HWID) с целью обхода банов. Читеры используют специальные утилиты для изменения серийных номеров дисков, MAC-адресов и т.д.

    🔹 DMA (Direct Memory Access)

    Прямой доступ к памяти, используемый в аппаратных читах (например, на основе FPGA). Позволяет читать память компьютера без использования ПО, что затрудняет детекцию.

    🔹 PE-файл (Portable Executable)

    Формат исполняемых файлов в Windows (.exe, .dll, .sys). Содержит заголовки, секции кода и данных, таблицы импорта/экспорта.

    🔹 Цифровая подпись

    Электронная метка, подтверждающая авторство и целостность файла. Проверяется через сертификаты (например, от Microsoft, Valve). Подозрительно, если подпись отсутствует, недействительна или выдана неизвестным издателем.

    🧠 Советы эксперта по форензике

    🔬 Анализ дампов памяти с помощью strings и YARA

    После снятия дампа процесса (например, cs2.exe или csrss.exe) используй утилиту strings (из Sysinternals) для извлечения всех текстовых строк. Затем прогони результат через набор YARA-правил, разработанных для обнаружения сигнатур читов.

    Пример команды:

    strings -n 8 cs2.dmp > strings.txt
        yara -r cheat_rules.yara cs2.dmp

    Ищи URL-адреса, пути к конфигам, названия проектов (например, "aimware", "neverlose").

    🛡️ Проверка драйверов и ядра

    Используй DriverView (NirSoft) или Autoruns (Sysinternals) для просмотра загруженных драйверов. Обрати внимание на:

    • Драйверы, подписанные неизвестными издателями.
    • Драйверы, расположенные в C:\Windows\Temp или пользовательских папках.
    • Наличие драйверов с именами, похожими на системные, но с нестандартным размером.

    Также проверь состояние Secure Boot и Hyper-V – отключение этих функций может указывать на использование низкоуровневых читов.

    🌐 Сетевой анализ

    Запусти netstat -anob и проверь, какие процессы открывают сокеты на нестандартные порты. Читы часто используют порты 4444, 5555, 6666, 8080 и т.п. для связи с сервером управления.

    Используй Wireshark для захвата трафика и поиска пакетов, содержащих игровые данные (например, координаты противников). Это требует глубокого анализа, но может выявить инжекты трафика.

    📋 Артефакты Windows: Amcache, Shimcache, SRUM

    Не ограничивайся только Prefetch и BAM. Проверь:

    • Amcache.hve – хранит информацию о запусках программ (путь, хеш, версия). Расположен в C:\Windows\AppCompat\Programs\Amcache.hve.
    • Shimcache (AppCompatCache) – записывает время запуска .exe, даже если файл удалён. Извлекается через Reg Query или утилиты.
    • SRUM (System Resource Usage Monitor) – ведёт журнал использования приложений и сети. Можно проанализировать через SRUM Dump.

    Эти артефакты могут дать информацию о запущенных файлах, которые уже были удалены с диска.

    🎯 Работа с MFT и USN Journal

    Помимо JournalTrace, используй MFT Explorer или NTFS Log Tracker для прямого анализа MFT. Ищи:

    • Файлы с временными метками, не соответствующими системным событиям.
    • Имена файлов, содержащие tmp, dat, cfg в подозрительных папках.
    • Скрытые потоки данных (ADS) – читы могут прятать конфиги в альтернативных потоках.

    Для проверки ADS используй dir /r или инструмент Streams из Sysinternals.

    🧩 Список популярных читов для поиска по сигнатурам

    При поиске по файловой системе и реестру используй следующие названия (неполный список):

    • Aimware – один из старейших платных читов.
    • Neverlose – популярный чит с системой подписки.
    • Vape – известен своим обфускатором.
    • Fatality – высокоуровневый чит для CS:GO/CS2.
    • Osiris – бесплатный опенсорсный чит.
    • Nixware – ещё один платный чит.
    • Gamesense – премиум-решение.
    • Rage – часто используется в названиях конфигов.

    Также ищи фрагменты строк: inject, overlay, esp, triggerbot, rcs.
