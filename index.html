📋 Общий гайд по проверке на читы CS2

Основные положения

  1. Используем AnyDesk или RustDesk для удалённого доступа.
  2. Скачиваем все программы только с официальных сайтов. Не предлагайте свои версии.
  3. Создайте отдельную папку для инструментов проверки.
  4. Игнорируйте скрытый рабочий стол и доп. мониторы. Работайте только с основным.
  5. За читы в CS:GO не баним. Проверяем только CS2.

Типы доказательств

  1. Хранение — файл найден в файловой системе.
  2. Сокрытие — логи удаления/переименования.
  3. Запуск — следы запуска .exe/.dll в памяти или Prefetch (.pf).
Правило выдачи наказания:
Чит должен быть либо запущен в течение последних 3 месяцев, либо храниться на ПК. Достаточно признания или характерных признаков (размер, цифровая подпись, троллинг папка и т.д.).

Если нашли подозрительный запуск, но файл отсутствует

1. Ищите по названию в Everything.
2. Если не нашли — JournalTrace и ищите действия с этим файлом.

TIER 3 Базовая проверка

Критерии TIER 3

  • ✅ Проверка CS2 по фасту (инвентарь, оверлей и т.д.)
  • ✅ Проверка стандартных папок
  • ✅ Использование всех базовых программ
  • ✅ Базовый анализ JournalTrace
  • ✅ Проверка папки CS2

Места для быстрой проверки

Загрузки, C:\, AppData, Program Files, папки игры CS2 (на Lua-скрипты, подозрительные DLL и .exe).

Типичные пути:
C:\Users\%USERNAME%\AppData\
C:\Program Files (x86)\Steam\steamapps\common\Counter-Strike Global Offensive\
C:\ProgramData\

🔍 Everything

Быстрый поиск по файловой системе. ⓘ

ext:.exe|ext:.dll

ПКМ по колонкам → Data Accessed (дата изменения) → сортировка по дате доступа.

Ctrl+F — быстрый поиск в текущей папке

📜 JournalTrace

  • Drive → Select → C:\
  • Drive → Scan
  • Layout → Data Grid / Directory Tree (в зависимости от нужды, ниже способ для Data Grid)
  • Поиск: .exe или .dll

Поиск удалённых файлов

  • Сортировка по Date (два клика)
  • Включить фильтр Deleted
  • Ищем подозрительные удаления после запуска CS2
  • ПКМ → Show file information
Удаление через Корзину:
Файлы попадают в C:\$Recycle.Bin и переименовываются ($Rxxxxxx). Ищите файлы, начинающиеся на $.

Поиск замены файлов

  • Сортировка по Date
  • Фильтр Rename: old name
  • Анализ через Show file information

TIER 2 Глубокая проверка

Критерии TIER 2

  • ✅ Всё из TIER 3
  • ✅ Анализ памяти (System Informer)
  • ✅ Расширенный поиск в JournalTrace с regex
  • ✅ Проверка реестра Windows
  • ✅ Анализ автозагрузки

Everything и LastActivity — вспомогательные. Основной акцент на продвинутых программах.

🖥️ System Informer

Установка: Включите "Enable kernel-mode driver" в System → Options. При ошибке — Help → Check for Updates → Canary.

Проверка через csrss.exe

  1. Нажмите сочетание клавиш ctrl + f
  2. Введите специализированный список для этого способа
  3. Обязательно включите RegEx (между Clear Search и Match Case)
  4. Нажмите "Искать" (Find)
  5. Просматривайте странные процессы

📜 JournalTrace (Advanced)

Используйте модифицированную версию с поддержкой regex. Скрыть файлы практически невозможно.

Совет: || как разделитель для фильтрации:

.exe||.dll||cheat||hack

Типы доказательств

🧠 Анализ CSRSS (csrss.exe)

csrss.exe — системный процесс Windows (Client Server Runtime Subsystem). Он логирует все возможные запуски программ в текущей сессии. Анализ его памяти позволяет обнаружить следы исполнения .exe и .dll файлов, даже если сами файлы уже удалены.

🔹 Анализ для .exe файлов

  1. В System Informer выберите процесс csrss.exe с меньшим размером в байтах.
  2. Примените фильтр по regex'у: 
    ^[A-Z]:\\.+\.(exe)$
  3. Создайте папку с любым удобным названием и сохраните дамп найденных строк.
  4. Проанализируйте полученные данные с помощью Paths Parser.

🔹 Анализ для .dll файлов

  1. Выберите процесс csrss.exe с большим размером в байтах.
  2. Примените фильтр по regex'у: 
    ^[A-Z]:\\.+\.(dll)$
  3. Сохраните дамп и проанализируйте через Paths Parser.

🔹 Обнаружение .dll и .exe одновременно

Для отображения обоих типов файлов в одном результате используйте универсальный regex:

^[a-zA-Z]:\\.+(\\)?.*?$

🔍 Поиск файлов с изменёнными расширениями

Для выявления файлов, у которых расширение было переименовано:

^[a-z]:.+\.((?!exe|pyd|manifest|dll|config|\\|cpl|microsoft-|shell).)*$

✅ Чек-лист проверки

Отмечайте выполненные пункты. Прогресс сохраняется.

📋 Сценарии проверки

⚡ Экспресс-проверка (10–15 минут)

Когда нужно быстро принять решение о допуске к игре.

  1. Визуальный осмотр — проверка рабочего стола, открытых окон, папки "Загрузки".
  2. Everything — поиск по ext:.exe|ext:.dll сортировка по дате доступа / размеру
  3. Prefetch — быстрый просмотр через WinPrefetchView на предмет недавно запущенных подозрительных .exe.
  4. Процессы — проверка диспетчера задач на аномальные имена процессов (например, csgo.exe в неправильной папке).
  5. Реестр — быстрая проверка HKEY_CURRENT_USER\Software на ключи с названиями читов.

Результат: если ничего не найдено — допуск. Если есть сомнения — переходим к полной проверке.

🔬 Полная проверка (1–2 часа)

Для сложных случаев или при подозрении на продвинутые читы.

  1. Всё из экспресс-проверки.
  2. JournalTrace — детальный анализ удалений и переименований за последние 30 дней.
  3. System Informer — анализ csrss.exe и других процессов на предмет строк-путей к .exe/.dll.
  4. ShellBagAnalyzer — проверка открытых папок (включая удалённые).
  5. BAM Parser — извлечение истории запусков из реестра.
  6. UserAssistsView — дополнительные следы запусков через оболочку.
  7. Проверка автозагрузки (shell:startup, задачи планировщика).
  8. Проверка сетевых соединений (netstat -ano) на предмет связи с внешними серверами.

Результат: полный отчёт с найденными уликами или их отсутствием.

⚠️ Проверка после бана

Когда пользователь уже получил бан и утверждает, что он «чистый».

  1. Сбор максимального количества данных — все логи, дампы памяти, скриншоты.
  2. Анализ временных меток — сопоставить время бана с активностью в системе (запуски, изменения файлов).
  3. Поиск следов античита — например, логи VAC или Faceit AC, если они есть на диске.
  4. Проверка на спуфинг — поиск программ для подмены HWID (например, spoofer.exe, driver.sys).
  5. Анализ скрытых процессов — с помощью GMER или аналогичных утилит.

Результат: заключение о наличии или отсутствии доказательств, подтверждающих бан.

🚩 Красные флаги (признаки возможных читов)

📁 Файловая система

  • 🔴 Файлы с названиями, содержащими cheat, hack, aim, wall, inject.
  • 🔴 Странные файлы с расширениями .exe или .dll, расположенные в Temp, AppData\Local\Temp, C:\Users\Public.
  • 🔴 Наличие упаковщиков (UPX, VMProtect) в исполняемых файлах (можно проверить через PE-bear или Detect It Easy).
  • 🔴 Файлы с поддельными цифровыми подписями (подпись присутствует, но недействительна или выдана неизвестным издателем).
  • 🔴 Конфигурационные файлы читов (.cfg, .ini, .json) с подозрительными названиями.

🧠 Память и процессы

  • 🔴 Процессы с нестандартными именами, например csgo.exe в папке AppData.
  • 🔴 Процессы, запущенные из папок без прав на запись (например, System32) с подозрительными именами.
  • 🔴 В памяти csrss.exe найдены пути к .exe или .dll, которые не являются системными.
  • 🔴 В памяти игрового процесса (cs2.exe) обнаружены строки с названиями читов или URL-адресами для связи с сервером читов.

📜 Логи и артефакты

  • 🔴 В JournalTrace зафиксированы массовые удаления .exe или .dll сразу после запуска CS2.
  • 🔴 В Prefetch есть записи о запуске подозрительных .exe, которые сейчас отсутствуют на диске.
  • 🔴 В реестре есть ключи с именами читов (например, Software\Aimware, Software\Vape).
  • 🔴 В ShellBag есть посещённые папки, ведущие на сайты с читами или на папки с названиями типа cheats, injects.
  • 🔴 В BAM Parser есть записи о запуске файлов, которые уже удалены с диска.

🌐 Сеть

  • 🔴 Активные сетевые соединения на нестандартные порты (например, 27015, 27016 — игровые, но если есть на 4444, 1337, 5555 — подозрительно).
  • 🔴 Наличие в памяти строк с IP-адресами, не относящимися к игровым серверам.

📖 Глоссарий терминов

🔹 USN Journal (Update Sequence Number Journal)

Журнал изменений файловой системы NTFS. Фиксирует все изменения (создание, удаление, переименование, изменение атрибутов). Используется для восстановления истории файлов даже после их удаления.

🔹 MFT (Master File Table)

Главная таблица файлов в NTFS. Содержит метаданные каждого файла и папки на томе. Анализ MFT позволяет находить скрытые или удалённые записи.

🔹 ShellBags

Записи в реестре Windows, хранящие историю открытия папок через проводник (размер окна, положение, представление). Могут сохранять информацию о папках, которые уже удалены.

🔹 Prefetch

Механизм Windows для ускорения загрузки приложений. Хранит информацию о запущенных .exe (путь, количество запусков, дата, загруженные модули). Находится в C:\Windows\Prefetch\*.pf.

🔹 BAM (Background Activity Moderator)

Артефакт реестра (HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\bam\UserSettings). Хранит время последнего запуска каждого .exe, даже если файл уже удалён.

🔹 CSRSS (Client Server Runtime Subsystem)

Системный процесс Windows, отвечающий за управление консольными окнами и завершением процессов. В его памяти могут оставаться строки с путями к запущенным приложениям.

🔹 Инжект (Code Injection)

Внедрение кода (обычно DLL) в адресное пространство другого процесса. Читы часто используют инжект для модификации игровой памяти.

🔹 Спуфинг (Spoofing)

Подмена аппаратных идентификаторов (HWID) с целью обхода банов. Читеры используют специальные утилиты для изменения серийных номеров дисков, MAC-адресов и т.д.

🔹 DMA (Direct Memory Access)

Прямой доступ к памяти, используемый в аппаратных читах (например, на основе FPGA). Позволяет читать память компьютера без использования ПО, что затрудняет детекцию.

🔹 PE-файл (Portable Executable)

Формат исполняемых файлов в Windows (.exe, .dll, .sys). Содержит заголовки, секции кода и данных, таблицы импорта/экспорта.

🔹 Цифровая подпись

Электронная метка, подтверждающая авторство и целостность файла. Проверяется через сертификаты (например, от Microsoft, Valve). Подозрительно, если подпись отсутствует, недействительна или выдана неизвестным издателем.
